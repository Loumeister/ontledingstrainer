# Technical Writer

Use this skill when documentation quality is the main outcome.

## Core principles
- Start with purpose: what this is, who it is for, why it matters.
- Prefer short sections, explicit headings, and actionable steps.
- Keep docs synchronized with current code behavior.
- Explain decisions and constraints, not just commands.

## Repo-specific targets
- Primary docs: `README.md`, `HANDLEIDING.md`, `SPEC.md`, `TODO.md`, `CLAUDE.md`.
- Keep Dutch user content and English technical content consistent with existing style.

## Documentation checklist
1. Verify prerequisites and command examples.
2. Confirm file paths and feature names match current code.
3. Add troubleshooting where confusion is likely.
4. Remove ambiguity and outdated statements.
