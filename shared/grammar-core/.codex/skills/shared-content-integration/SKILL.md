# Shared Content Integration

Read first:
- `docs/content-authoring-rules.md`
- `docs/repo-sync-strategy.md`
- `docs/taxonomy-governance.md`

Use when:
- integrating shared sentence content into a product repo
- designing adapters from shared objects to product-specific content
- reviewing whether shared content stays canonical

Rules:
- keep integration explicit and lightweight
- preserve canonical content in grammar-core
- let product repos adapt shared content without rewriting the shared source silently
- prevent schema drift and taxonomy drift
